# v0.1.0-alpha

System Coach and Maintenance Manager is ready for public alpha evaluation.

This release is meant for people who want a local-first, supervised way to inspect a machine, understand the installed development stack, prepare maintenance plans, and review guarded actions before anything changes.

## What Works

- Browser fallback mode for portable first-run use.
- Native GTK desktop shell on Linux systems with the required GTK bindings.
- Local Review for installed tool and stack discovery.
- Capability Profile for OS, desktop, package-manager, model-runtime, privilege-helper, and documentation fit.
- Read-only Maintenance Diagnostics.
- Request Desk planning with command previews, risk notes, rollback notes, and approval gates.
- Server-side action plan registry with short-lived `plan_id` execution.
- Exact confirmation phrase enforcement for guarded actions.
- Pop!_OS + COSMIC deep scan, research records, local-model analysis, fix-plan preparation, verification, and lesson capture.
- Model Provider Setup for local Ollama, bring-your-own-key cloud readiness, and deterministic fallback.
- Local maintenance history stored on the user's machine.

## Safety Boundaries

- The app does not silently repair a computer.
- The browser cannot submit an executable action contract for the server to trust.
- Model output can explain, classify, and reason, but deterministic guarded planners choose executable commands.
- Raw API keys are not accepted or stored.
- Cloud model use requires the user's own environment variable and explicit provider settings.
- Elevated actions still require operating-system approval.
- Destructive cleanup, broad deletes, firmware work, OS refreshes, and guessed service restarts remain blocked unless a future guarded contract supports them.

## Public-Use Caveats

- This is alpha software. Review plans before approval and keep backups for important systems.
- Run browser fallback first on unfamiliar machines.
- Do not share raw logs, `history/*.jsonl`, API keys, tokens, private paths, hostnames, serial numbers, or screenshots with personal information in public issues.
- Pop!_OS/COSMIC support is strongest on matching Pop!_OS + COSMIC systems. Other Linux desktops should expect advisory behavior where platform commands differ.
- Local model behavior depends on the models installed through Ollama. Deterministic fallback remains available when no model is ready.

## Recommended First Run

```bash
PYTHONPATH=src python3 -m system_coach_maintenance_manager --browser
```

Then review:

1. Capability Profile
2. Model Provider Setup
3. Local Review
4. Maintenance Diagnostics
5. Request Desk or Pop!_OS + COSMIC Agent plans

## Validation

Release checks include governance preflight, unit tests, compile checks, browser JavaScript syntax, whitespace checks, secret-pattern scanning, and browser API smoke for `/health`, `/api/capabilities`, and `/api/model-provider`.
